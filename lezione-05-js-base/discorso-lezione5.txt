Allora ragazzi, oggi cambiamo marcia. Fino ad ora abbiamo lavorato con HTML e CSS: HTML per costruire lo scheletro della pagina, titoli, paragrafi, immagini, form; CSS per vestirla, dargli colori e spaziature, renderla bella da vedere. Ma finora la pagina era "statica": non succedeva niente, non reagiva a quello che l'utente faceva. Oggi vediamo JavaScript, il linguaggio che dà vita alla pagina: fa comparire un messaggio quando clicchi un bottone, controlla se hai riempito bene un form, aggiorna un carrello della spesa senza ricaricare la pagina. In pratica: HTML è il corpo, CSS sono i vestiti, e JavaScript sono i muscoli, quello che fa muovere le cose.

Partiamo dalle basi: le variabili. Una variabile è un contenitore dove salviamo un valore, per riusarlo più avanti. In JavaScript abbiamo due modi per crearla: "let" e "const". La differenza è semplice: "let" la puoi cambiare dopo, "const" no, una volta assegnato il valore resta per sempre. Esempio concreto: "eta", l'età di una persona, la scrivo con "let" perché ogni compleanno cambia: let eta = 16, e l'anno prossimo eta = 17. Invece "nomeScuola" lo scrivo con "const", perché per tutto l'anno non cambia: const nomeScuola = "Liceo Italia". Consiglio pratico: usate sempre const di default, e passate a let solo quando sapete già che quel valore dovrà cambiare, tipo un contatore o un punteggio.

Ora i costrutti che ci permettono di prendere decisioni e ripetere azioni. Il primo è if/else, che funziona come nella vita reale: "se piove, prendo l'ombrello, altrimenti esco senza". In codice: if (piove) { prendiOmbrello() } else { esci() }. Esempio scolastico: se il voto è maggiore o uguale a sei, è sufficiente, altrimenti no. Semplice: è una condizione, e in base a se è vera o falsa il computer fa una cosa o un'altra.

Poi ci sono i cicli, che ripetono un'azione senza riscrivere il codice cento volte. Il "for" si usa quando sapete già QUANTE volte ripetere una cosa: devo salutare cinque persone in fila, so già che sono cinque, quindi uso un for da 1 a 5. Il "while" invece si usa quando non sapete in anticipo quante volte serve ripetere, ma sapete QUANDO fermarvi: sto svuotando un serbatoio di benzina un litro alla volta, e continuo finché non è vuoto. Non so a priori quanti litri ci sono, ma so la condizione per fermarmi: benzina maggiore di zero.

Passiamo alle funzioni. Una funzione è un pezzo di codice che scriviamo una volta e richiamiamo tutte le volte che vogliamo, magari con dati diversi ogni volta. Oggi vediamo soprattutto le funzioni freccia, il modo moderno e più usato di scriverle. Invece di "function saluta(nome) { return ... }", scriviamo "const saluta = (nome) => { return ... }". È la stessa identica cosa, cambia solo la sintassi, e se la funzione è corta la scriviamo anche su una riga sola: const saluta = nome => `Ciao ${nome}`. Abituatevi a questa forma perché la trovate ovunque nel codice moderno.

Adesso il pezzo forte della lezione: gli array. Un array è una lista ordinata di valori, tipo la lista della spesa, o l'elenco degli studenti della classe. Ogni elemento ha una posizione, che si chiama indice, e attenzione: si parte da zero, non da uno. Il primo elemento è in posizione 0, il secondo in posizione 1, e così via. Gli array sono ovunque in programmazione perché quasi sempre lavoriamo con gruppi di dati, non con un dato singolo.

Per lavorare con questi gruppi di dati, JavaScript ci dà un set di metodi che dovete assolutamente conoscere. ForEach prende ogni elemento dell'array e ci fa fare qualcosa, tipo stamparlo: "per ognuno di voi, fai questa cosa". Map prende ogni elemento, lo trasforma, e crea un nuovo array con i risultati: prendo una lista di numeri e li raddoppio tutti, ottengo una nuova lista. Filter sceglie solo gli elementi che rispettano una condizione, crea un nuovo array più corto: prendo solo i voti sufficienti da una lista di voti. Find è simile a filter ma si ferma al primo elemento che trova, e restituisce solo quello, non una lista. Sort ordina l'array, dal più piccolo al più grande o come decidiamo noi. E infine reduce, il più potente: prende tutto l'array e lo "riduce" a un unico valore, per esempio la somma di tutti i numeri, o la media di tutti i voti.

Ultimo argomento di oggi: i dizionari, che in JavaScript si chiamano oggetti. Se l'array è una lista con posizioni numeriche, il dizionario è una lista con dei nomi: invece di dire "l'elemento in posizione 0", dico "il valore alla chiave nome". Per esempio uno studente lo posso rappresentare così: { nome: "Luca", cognome: "Verdi", eta: 16 }. Per leggere un valore ho due modi: studente.nome, col punto, oppure studente["nome"], con le parentesi quadre e le virgolette. Sono equivalenti, ma le parentesi quadre servono soprattutto quando la chiave ce l'ho dentro una variabile. E se voglio scorrere TUTTE le chiavi di un dizionario, uso un ciclo speciale chiamato for...in, che mi dà una alla volta tutte le chiavi, e da lì posso leggere anche il valore corrispondente.

Bene ragazzi, oggi abbiamo messo le basi vere e proprie della programmazione: variabili, condizioni, cicli, funzioni, array e dizionari. Sono gli ingredienti che userete sempre, in ogni progetto. Nella prossima lezione vedremo come usare questi strumenti su dati reali: per esempio leggere un file CSV e trasformarlo in un array di dizionari, oppure collegare JavaScript alla pagina HTML per far reagire davvero il sito a quello che l'utente fa, toccando quello che si chiama il DOM. Quindi tenete bene a mente quello che abbiamo visto oggi, perché da qui in poi lo useremo in continuazione.
